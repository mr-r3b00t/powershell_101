Introduction to PowerShell

Context: PowerShell is a powerful scripting language and command-line shell designed for system administration and automation, built on .NET.

Description: Learn what PowerShell is, its history, and how it integrates with Windows and cross-platform systems. Understand the difference between PowerShell and traditional command-line interfaces.

Key Learning Outcome: Understand the purpose and basic capabilities of PowerShell.

Learn More
×

Extended Reading: Introduction to PowerShell

PowerShell, developed by Microsoft, combines the speed of a command-line interface with the flexibility of a scripting language. Released in 2006 as Windows PowerShell, it became open-source in 2016 as PowerShell Core, extending support to Linux and macOS. Built on the .NET framework, it allows seamless interaction with Windows components like the Registry, WMI, and COM objects.

Unlike traditional shells like CMD, PowerShell operates on objects, not just text, enabling richer data manipulation. For example, running `Get-Process` returns a collection of process objects with properties like Name, ID, and CPU usage, which can be filtered or sorted directly.

The PowerShell Gallery (powershellgallery.com) is a key resource for finding community-created scripts and modules. Try running `Install-Module -Name PSReadLine` to enhance your PowerShell console with features like syntax highlighting and auto-completion.

Description: Learn the structure of cmdlets (verb-noun format, e.g., Get-Help) and how to use them to perform tasks like retrieving system information.

Key Learning Outcome: Execute basic cmdlets to interact with the system.

Learn More
×

Extended Reading: PowerShell Cmdlets

Cmdlets follow a verb-noun naming convention (e.g., `Get-Process`, `Set-Location`) to ensure consistency and predictability. The verb indicates the action (e.g., Get, Set, New), and the noun specifies the target (e.g., Process, Service). This structure makes cmdlets intuitive to discover and use.

Use `Get-Command` to list all available cmdlets, or filter by verb or noun, like `Get-Command -Verb Get`. For detailed help, run `Get-Help -Full`, such as `Get-Help Get-Process -Full`, to see parameters, examples, and descriptions.

Try practical examples: `Get-Service | Where-Object {$_.Status -eq 'Running'}` lists running services, or `Stop-Process -Name notepad` terminates Notepad. Cmdlets are lightweight .NET classes, so they’re efficient and extensible, forming the backbone of PowerShell automation.

Extended Reading: Variables and Data Types

PowerShell variables are declared with a $ prefix, like `$count = 42` or `$greeting = "Hello"`. PowerShell is loosely typed, so you don’t need to declare a type upfront, but you can enforce types using casts, e.g., `[int]$count = 42` or `[string]$text = "123"`. This ensures `$text` is treated as a string, not a number.

Common data types include integers (`[int]`), strings (`[string]`), arrays (`$list = 1,2,3`), and hashtables (`$dict = @{Name="John"; Age=30}`). Arrays are versatile; access elements with `$list[0]` or add items with `$list += 4`. Hashtables are key-value pairs, useful for structured data, accessed like `$dict.Name`.

Scope is also key: variables are local by default, but you can use `global:`, `script:`, or `private:` prefixes to control scope. For example, `$global:setting = "value"` makes a variable accessible across scripts. Experiment with `$array = Get-Process | Select-Object -First 3` to store the first three processes in an array.

Key Learning Outcome: Use operators to perform calculations and comparisons.

Learn More
×

Extended Reading: Operators

PowerShell supports arithmetic operators like `+` (addition), `-` (subtraction), `*` (multiplication), `/` (division), and `%` (modulus). For example, `$result = 10 + 5 * 2` yields 20, following standard precedence. Use parentheses to control order: `(10 + 5) * 2` gives 30.

Comparison operators include `-eq` (equals), `-ne` (not equals), `-gt` (greater than), `-lt` (less than), `-ge`, and `-le`. They’re case-insensitive by default; use `-ceq` for case-sensitive equality. For instance, `"hello" -eq "HELLO"` is true, but `"hello" -ceq "HELLO"` is false. String matching is also supported with `-like` (e.g., `"file.txt" -like "*.txt"`) and `-match` for regex.

Logical operators `-and`, `-or`, and `-not` combine conditions. Example: `if ($a -gt 5 -and $b -lt 10) { Write-Output "Valid" }`. Try this: `$processes = Get-Process | Where-Object { $_.CPU -gt 100 -and $_.Name -like "chrome*" }` to filter high-CPU Chrome processes.

Extended Reading: Conditional Statements

The `if` statement evaluates a condition: `if ($count -gt 10) { Write-Output "Count is high" }`. Add `elseif` for additional conditions and `else` for a default: `if ($count -gt 10) { "High" } elseif ($count -lt 5) { "Low" } else { "Medium" }`. Conditions can include any expression that evaluates to true or false.

The `switch` statement is an alternative for multiple conditions, improving readability. Example: `switch ($value) { 1 {"One"} 2 {"Two"} default {"Other"} }`. It supports regex and wildcard matching, like `switch -Wildcard ($file) { "*.txt" {"Text file"} "*.exe" {"Executable"} }`.

Practical example: `Get-Service | ForEach-Object { if ($_.Status -eq 'Running') { Write-Output "$($_.Name) is running" } else { Write-Output "$($_.Name) is stopped" } }`. This checks the status of all services. Nesting `if` statements is possible but can reduce readability; consider `switch` for complex logic.

Key Learning Outcome: Use loops to automate repetitive tasks.

Learn More
×

Extended Reading: Loops

The `foreach` loop is ideal for arrays or collections: `foreach ($item in $array) { Write-Output $item }`. It’s commonly used with cmdlets, like `Get-Process | ForEach-Object { Write-Output $_.Name }`. The `ForEach-Object` cmdlet is similar but pipeline-friendly.

The `for` loop is great for indexed iteration: `for ($i = 0; $i -lt 5; $i++) { Write-Output $i }`. The `while` loop runs as long as a condition is true: `while ($count -lt 10) { $count++; Write-Output $count }`. The `do-while` loop ensures at least one execution: `do { $count--; Write-Output $count } while ($count -gt 0)`.

Use `break` to exit a loop early and `continue` to skip to the next iteration. Example: `Get-ChildItem *.log | ForEach-Object { if ($_.Extension -ne ".txt") { continue } Write-Output $_.Name }` processes only text files. Loops are powerful for automation, like restarting stopped services: `Get-Service | Where-Object {$_.Status -eq 'Stopped'} | ForEach-Object { Start-Service $_.Name }`.

Extended Reading: Functions

Define a function with the `function` keyword: `function Get-Square ($num) { return $num * $num }`. Call it with `Get-Square 5`, which returns 25. Functions can have multiple parameters: `function Add-Numbers ($a, $b) { return $a + $b }`. Use default values for flexibility: `function Greet ($name = "User") { "Hello, $name!" }`.

Functions support advanced features like parameter validation: `function Set-Age ([ValidateRange(0,120)][int]$age) { $age }` ensures `$age` is between 0 and 120. Use `[CmdletBinding()]` for cmdlet-like behavior, enabling parameters like `-Verbose` or `-ErrorAction`.

Practical example: `function Get-RunningServices { Get-Service | Where-Object {$_.Status -eq 'Running'} | Select-Object Name, DisplayName }`. Call it with `Get-RunningServices` to list running services. Functions can be stored in scripts or modules for reuse across projects, improving code organization and maintainability.

Error Handling

Context: Errors can disrupt scripts; handling them ensures robustness.

Description: Use try-catch blocks to manage errors and ensure scripts run smoothly.

Key Learning Outcome: Implement error handling to manage script failures.

Learn More
×

Extended Reading: Error Handling

Use `try { #code } catch { #handle error }` to manage errors. For example: `try { Get-Content "nonexistent.txt" } catch { Write-Output "File not found" }`. The `catch` block can access error details via `$_`, like `$_.Exception.Message` for the error message.

PowerShell distinguishes between terminating and non-terminating errors. Cmdlets like `Get-Content` produce non-terminating errors by default, so use `-ErrorAction Stop` to make them terminating: `try { Get-Content "nonexistent.txt" -ErrorAction Stop } catch { "Error: $($_)" }`. Multiple `catch` blocks can handle specific exceptions: `catch [System.IO.FileNotFoundException] { "File missing" }`.

The `finally` block runs regardless of errors: `try { #code } catch { #handle } finally { Write-Output "Cleanup done" }`. Example: `try { Stop-Process -Name "notepad" -ErrorAction Stop } catch { Write-Output "Error stopping process: $($_)" } finally { Write-Output "Attempt completed" }`. This ensures robust scripts, especially for automation tasks.

Extended Reading: Working with Files

Use `Get-Content` to read file contents: `$text = Get-Content "file.txt"` loads the file into a variable, with each line as an array element. Write to files with `Set-Content`: `Set-Content "file.txt" "Hello, World!"` overwrites the file, while `Add-Content` appends: `Add-Content "file.txt" "New line"`.

File management cmdlets include `New-Item` to create files or directories (`New-Item -Path "newfile.txt" -ItemType File`), `Remove-Item` to delete (`Remove-Item "oldfile.txt"`), and `Move-Item` to rename or move (`Move-Item "file.txt" "backup/file.txt"`). Use `Test-Path` to check existence: `if (Test-Path "file.txt") { Write-Output "File exists" }`.

Practical example: `Get-ChildItem *.log | ForEach-Object { $content = Get-Content $_.Name; if ($content -match "ERROR") { Write-Output "$($_.Name) contains errors" } }` scans log files for errors. For large files, use `Get-Content -ReadCount 0` to read efficiently or `-Tail 10` for the last 10 lines.

Piping and Redirection

Context: PowerShell’s pipeline passes data between cmdlets for efficient workflows.

Description: Understand piping (|) and redirection (> or >>) to chain commands and save output.

Key Learning Outcome: Use piping and redirection to streamline tasks.

Learn More
×

Extended Reading: Piping and Redirection

The pipeline (`|`) passes objects from one cmdlet to another, enabling powerful workflows. For example, `Get-Process | Where-Object { $_.CPU -gt 100 } | Sort-Object CPU -Descending` filters processes using over 100 CPU units and sorts them. Unlike text-based shells, PowerShell pipes structured objects, preserving properties for further manipulation.

Redirection operators include `>` (overwrite file), `>>` (append to file), and `2>` (redirect errors). Example: `Get-Process > processes.txt` saves output to a file, while `Get-Content "nonexistent.txt" 2> errors.txt` redirects errors. Use `Out-File` for more control: `Get-Service | Out-File "services.txt" -Encoding utf8`.

Practical pipeline: `Get-ChildItem *.txt | ForEach-Object { $_.Name | Out-File "filelist.txt" -Append }` creates a list of text file names. Combine piping and redirection: `Get-Process | Where-Object {$_.WorkingSet64 -gt 100MB} | Select-Object Name, WorkingSet64 | Out-File "highmem.txt"`. This filters processes using over 100MB of memory and saves their details.

Extended Reading: Modules

Modules are packages of cmdlets, functions, and scripts that extend PowerShell. Use `Import-Module ActiveDirectory` to manage Active Directory objects or `Import-Module SqlServer` for SQL Server tasks. List available modules with `Get-Module -ListAvailable` and install new ones from the PowerShell Gallery: `Install-Module -Name Az -Scope CurrentUser` for Azure management.

Modules can be script-based (.psm1) or binary (.dll). Create a simple module by saving functions in a `.psm1` file, like `function Get-CustomData { #code }`, and import it with `Import-Module ./MyModule.psm1`. Use `Export-ModuleMember` to specify which functions are public. Modules support versioning, ensuring compatibility: `Import-Module Az -RequiredVersion 9.0`.

Example: After installing `PSWindowsUpdate`, run `Get-WUList` to list Windows updates. For automation, use `Find-Module *Security* | Where-Object {$_.Author -eq "Microsoft"}` to discover Microsoft-authored security modules. Modules are essential for scaling PowerShell tasks, from system administration to cloud management.

Extended Reading: Scripting Basics

PowerShell scripts are saved with a `.ps1` extension and executed with `.\script.ps1`. Set the execution policy if restricted: `Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned`. A basic script might look like: `# Backup Files; Get-ChildItem *.txt | Copy-Item -Destination "backup/"`.

Structure scripts with comments (`#` or `<# #>` for blocks), functions, and error handling. Example: `<# Monitor Services #> function Check-Service { param($name) try { $svc = Get-Service $name -ErrorAction Stop; Write-Output "$name is $($svc.Status)" } catch { Write-Output "Error: $_" } } Check-Service "wuauserv"`. This script checks the Windows Update service status.

Scripts can accept parameters: `param ($Path) Get-ChildItem $Path | Out-File "filelist.txt"`. Run it with `.\script.ps1 -Path "C:\Logs"`. For scheduling, use Task Scheduler to run scripts periodically. Combine cmdlets, loops, and functions for automation, like a script to archive old logs: `Get-ChildItem *.log | Where-Object {$_.LastWriteTime -lt (Get-Date).AddDays(-30)} | Move-Item -Destination "archive/"`.
